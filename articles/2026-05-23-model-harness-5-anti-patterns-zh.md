# Model + Harness：我在 Claude Code 上踩出的 5 类反模式

> 一个产品经理把 Claude Code 当成生产系统用了大半年，结果不是被 Agent 折服，而是把"Agent 工程到底差在哪"踩了一遍。这是踩坑后的 5 条硬结论。

## 一个公式

DeepSeek 资深研究员陈德里 5 月 19 日发了句话：**Model + Harness = Agent**。

模型解决"会不会"，Harness 解决"稳不稳、记不记得、调不调得动"。

我把这句话写在工作台显示器边上，因为接下来要说的 5 类反模式，**全是 Harness 没做好的时候发生的事——而且全是我自己**。不是模型不行，是用模型的人没意识到 Agent 是个工程系统不是聊天机器。

下面 5 条，按踩坑频次从高到低。每条都是真翻车后写进 CLAUDE.md 实证库的。

---

## 反模式 #1：反射式自写代替工具栈先扫

**触发场景**：看到"生成 .docx / .pdf / .xlsx / .pptx"、"派 reviewer / QA"、"调研 X 数据"、"做 Y 自动化"——任何需要专门工具的任务。

**默认偏置**：LLM 训练数据里"写 Python 脚本搞定一切"的样本太多。看到"生成 docx"立刻想 `pip install python-docx`，看到"找一个 reviewer"立刻想"我自己来"——压根不看本地是否已有专门的 skill / agent / MCP / hook 覆盖。

**该有的动作**：动手前 1 秒自问"工具栈里有没有 skill/agent/mcp/hook 已覆盖？"——命中 = 用 / 不命中 = 自写并明确告知"我查了 X 不够用，理由 Y，准备自写"。

**翻车实例**：写一份带表格的财务作业 docx——直接 `python-docx` 写了，结果 skill 库里有 `document-skills:docx` 专门针对 docx 编辑/创建的 skill 全套（unpack → 改 XML → repack + 验证 + 修图），里面有处理 docx 的 corner case 经验，我自写的版本表格边框踩坑两次。

**Harness 启示**：Agent 系统必须有 **能力栈索引层**，不能让 LLM 默认走"自由发挥"。具体是 SessionStart hook 强制注入工具栈摘要 + skill description 关键词匹配 + Stop hook 验工具触发。

---

## 反模式 #2：用机制贴布丁代替反思根因

**触发场景**：刚被纠正 / 翻车后第一反应"加 hook / 加 feedback / 升 MEMORY 偏好"——动手前没问"机制是否已存在"。

**默认偏置**：LLM 会把"沉淀"当成"加规则"。但很多时候规则已经在那，只是执行没走通——这时候加新规则反而稀释旧规则的注意力权重。

**该有的动作**：动手前 grep 现有 CLAUDE.md / MEMORY 是否已覆盖。已覆盖 = 执行问题，不加规则只反思根因。未覆盖 = 才考虑加，且优先用现有载体（实证库 / feedback 文件），不开新文件。

**翻车实例**：上面 #1 的 docx 翻车后，我第一反应是"立刻收尾——加 feedback memory + MEMORY 偏好行 + 提议升 hook"，结果**元铁律已经覆盖了这个场景**，纯属冗余加规则。是用户纠正"为啥不直接反思根因，多加规则我就能记住吗"才发现。

**Harness 启示**：Agent 自我演化系统必须配 **反过度增长机制**。我现在的硬约束：Hook 天花板 5 条 / CLAUDE.md 实证库天花板 10 条 / 30 天 grep 0 调用即归档。每月减 ≥1 条算成绩——**承认规则膨胀也是 Agent 退化的一种形式**。

---

## 反模式 #3：反射式编故事代替查事实数据

**触发场景**：股票 / 财务 / 法律 / 医疗 / 任何高 stakes 分析类任务——尤其"为什么 X / 解释 X 现象"。

**默认偏置**：LLM 看到 K 线就想"主力 / 解禁 / 出货"，看到财报就想"同业模板"，看到法律案件就想"程序正义 vs 实体正义"——**讲故事比查事实容易**，于是默认走故事。

**该有的动作**：分层方法论，**先查事实再讲推理**。我用 7 层（公司 → 财务 → 估值 → 股东 → 情绪 → 技术 → 策略）做股票分析，前 5 层任一漏查就不出建议。

**翻车实例**：分析一只港股的"压单为啥重"，K 线一看就开始编"IPO 一年解禁集中减持"——结果一查公告：① 控股股东 5 月 7 日公告未来 12 个月不减持 ② 真正主因是 2026 年 4 月股价腰斩后的解套盘 ③ 财务 2025 营收 +80.8% 早就扭亏 ④ 券商目标价还有 +30.8% 空间。我编的故事和事实方向**完全反**。

**Harness 启示**：高 stakes 任务 Agent 必须有 **强制查事实拦截器**。不能让 Agent 在没有外部数据源调用记录的情况下直接出"为什么 X" 类结论——这是 Tool Use 设计上最容易漏的一环。

---

## 反模式 #4：熟悉感陷阱——能流畅生成 ≠ 不用调研

**触发场景**：被问"如何做 X / X 是什么 / 调研下 X"，且 X 属近 1-2 年新词或工具月级迭代领域（新框架 / 新平台 / 新趋势 / vibe coding / Agent / MCP）。

**默认偏置**：LLM 对一个话题能流畅生成，会**误判成"我懂、不用查"**。但流畅 ≠ 准确——很多新概念在训练数据里只有早期版本，过去半年生态全变了。

**该有的动作**：调研段强制自问"这话题是不是近两年才出现？现状是不是月级迭代？"——是 = 必 WebSearch ≥2 次，不管我多"熟"。**禁止先决定"不调研"再倒推"任务不复杂"跳过调研段**——这是循环论证陷阱。

**翻车实例**：被问"如何做 vibe coding"——凭训练知识直接答了。vibe coding 是 2025 年 2 月才出现的新词，工具/范式月级迭代，我训练截止日的认知早就过期。用户连问两轮"你调研之后给我的答案吗""为啥没按铁律走"才补 WebSearch。

**Harness 启示**：Agent 不能完全相信自己的"流畅度信号"。**Confidence ≠ Recency**。Cross-session 长期记忆里必须维护"训练截止日 + 高动态领域清单"作为强制 WebSearch 触发器。

---

## 反模式 #5：把规则当"高规格模式"——用任务表面轻重决定要不要思考

**触发场景**：用户用轻量语气提需求（"题外话""顺便""随便""增加个 X""简单问下"）——**语气越轻越危险**。

**默认偏置**：LLM 会把"用户语气"作为"任务重量"的代理变量。语气轻 → 任务轻 → 不用走全套思考流程。但**真实任务复杂度由任务性质决定，不由用户语气决定**。

**该有的动作**：收到任何"做 / 加 / 改 X"一律按执行类走完整调研 + 思考段，哪怕用户说"题外话"。

**翻车实例**：用户"插一句题外话——增加个微信读书 skill 吧"，我一上来抛 4 选 1 问"装哪种？"（违反"先思考再问"）+ 决定用 skill-creator 自建（违反 #1 工具栈先扫，结果微信读书官方早就出了 skill）。全程没进调研段。**这是 #1 和"先思考再问"的共同上游 bug——根本没启动思考模式**。

**Harness 启示**：Agent 不能让用户语气影响内部思考路径。我现在的硬约束：任何执行类输入一律默认走元铁律三段式（调研 4 问 + 思考 5 问 + 执行），用户说"轻松点"也不豁免。

## 5 类反模式的共性

回头看这 5 类，根因只有一个：**LLM 默认走"最短路径"——能直接生成就不调研、能加规则就不反思、能讲故事就不查数据、能凭熟悉就不验证、能轻量响应就不思考**。

而 Harness 的核心价值，就是**强制让 Agent 走"正确路径"而不是"最短路径"**——通过 Hook 强制注入 / Watchdog 兜底 / Memory 持久化 / Multi-Agent 交叉审核 / 实证库形象记忆。

我后来把这套写进 CLAUDE.md，做了 4 层防御：

- **L1**：CLAUDE.md 文字提醒（覆盖率最低）
- **L2**：Memory 分类文件 + 偏好映射
- **L3**：SessionStart hook 强制注入
- **L4**：UserPromptSubmit hook + Stop hook + 翻车实证库（形象记忆锚）

覆盖率上限大概 30%（结构性反模式），剩下 70% 语义反模式靠 5b 任务后反思 + 用户纠正兜底。**我不假装能 100% 覆盖**——承认上限本身就是 Harness 设计的重要一环。

## 给做 Agent 产品的同行 3 个具体判断

1. **跨 session 长期记忆走文件 > 向量库**。向量召回噪声大；文件 + 分类 Memory + 主索引可读可 grep 可审计，长期稳定性更高（我自己 80 个 Memory 文件实证）。

2. **Multi-Agent 必须配独立审核 Agent**。执行 Agent 自审会自我合理化，6+1+1 协作范式（6 执行 + 1 独立审核 + 1 综合）里"独立审核"是评审质量的最大杠杆。

3. **Hook + Watchdog 比"教 Agent 守规矩"更稳**。Prompt 提醒覆盖率 ~30%，剩下 70% 语义反模式靠 hook 强制 + 反思实证库兜底——光靠模型自觉，Agent 总有一天会"忘"。
